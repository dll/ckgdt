# 答辩直播功能实现说明

## 概述

答辩直播是一个基于 HTTP MJPEG 的局域网多端协作流媒体系统，支持**教师主播**、**答辩学生**和**观众**三种角色。

---

## 架构

```
教师端(Windows)          答辩学生端(Win/Android/Web)       观众端(Win/Android/Web)
    ↓                           ↓                              ↓
启动 HTTP 服务器         推送屏幕+摄像头 JPEG           连接教师 IP 观看 MJPEG 流
(DefenseStreamingServer)  → POST /frame/{win|phone|camera}  ← GET /stream/feed
    ↓                           ↓                              ↓
端口 8766                  3秒心跳 POST /heartbeat         自动填充 IP（通知携带）
```

---

## 四种角色模式

### 1. Presenter（教师主播）
- **功能**：运行流媒体服务器，接收答辩学生推流，转发给观众
- **操作**：
  - 点击"开始直播"启动服务器（自动获取本机 LAN IP）
  - 可选：开启自己的桌面+摄像头作为辅助画面
  - 实时显示观众数量（通过 `/status` 轮询）
- **UI**：显示服务器 IP、端口、观众数、桌面/摄像头开关
- **代码**：`_initPresenter()` → `DefenseStreamingServer.start()`

### 2. Defender（答辩学生）
- **功能**：将本地屏幕+摄像头推送到教师服务器
- **平台支持**：
  - **Windows**：`WinScreenCapturer` 15fps + Camera 3fps
  - **Android**：`PhoneScreenCapturer` (MediaProjection 全屏) 10fps + Camera 3fps
  - **Web**：屏幕捕获 API + Camera（未来实现）
- **操作**：
  1. 从通知进入 → `serverIp` 自动填充
  2. 点击"开始答辩" → 自动启动推流
  3. 双分屏预览：左侧屏幕、右侧摄像头
  4. 点击"结束答辩"关闭
- **心跳**：每 3 秒 POST `/heartbeat` 携带 `{"deviceName":"答辩学生 2020211613","source":"defender"}`
- **代码**：`_initDefender()` → `_startWinToRemote()` / `_startCamToRemote()` + `_startHeartbeat()`

### 3. Viewer（观众）
- **功能**：连接教师服务器观看答辩直播
- **操作**：
  1. 输入教师 IP（或从通知自动填充）
  2. 点击"连接"
  3. 显示 MJPEG 合成画面
- **UI**：IP 输入框 + 全屏播放器（`DefenseViewerWidget`）
- **代码**：`_connectViewer()` → `http://{ip}:8766/stream/feed`

### 4. Phone（手机投屏）
- **功能**：学生手机屏幕投射到教师直播画面（辅助角色）
- **操作**：
  1. 输入教师 IP
  2. 点击"投屏" → MediaProjection 全屏捕获
  3. 推送到 `/frame/phone`
- **代码**：`_startPhoneShare()` → `PhoneScreenCapturer.start()`

## 通知+自动填充 IP 机制

### 流程
1. **教师授权**：`LiveAuthorizeSheet` → 勾选答辩学生 → 点击"保存授权"
2. **发送通知**：`NotificationService.notifyDefenseAuthorized(studentId, studentName, serverIp)`
   - `serverIp` = `NetworkUtils.getLocalIp()` (教师本机 LAN IP)
   - 通知类型 `type='defense'`，`action_data='{"serverIp":"192.168.x.x"}'`
3. **学生接收**：`NotificationListPage` 显示"开始答辩"按钮
4. **一键进入**：点击通知 → `DefenseBroadcastPage(initialRole:'defender', serverIp:serverIp)`
   - IP 自动填充到 `_ipCtrl`
   - `initState` 调用 `_initDefender()` 自动连接

### 关键代码
- **发送通知**：`lib/presentation/widgets/live_authorize_sheet.dart:108-130`
- **通知详情**：`lib/presentation/pages/notification/notification_list_page.dart:353-362`
- **跳转逻辑**：`lib/presentation/pages/notification/notification_list_page.dart:427-435`

## HTTP 服务器 API

### DefenseStreamingServer 端点

| 路径 | 方法 | 功能 | 参数 |
|------|------|------|------|
| `/status` | GET | 服务器状态（观众数、帧年龄） | - |
| `/heartbeat` | POST | 客户端心跳（10秒超时） | `{"deviceName":"...", "source":"..."}` |
| `/sessions` | GET | 所有活跃会话列表 | - |
| `/kick/{ip}` | POST | 踢出指定 IP | - |
| `/frame/win` | POST | 推送 Windows 桌面帧 | Body: JPEG bytes |
| `/frame/phone` | POST | 推送手机屏幕帧 | Body: JPEG bytes |
| `/frame/camera` | POST | 推送摄像头帧 | Body: JPEG bytes |
| `/stream/feed` | GET | MJPEG 合成流（观众） | - |
| `/raw/win` | GET | MJPEG 单路 Windows 流 | - |
| `/raw/phone` | GET | MJPEG 单路手机流 | - |
| `/raw/camera` | GET | MJPEG 单路摄像头流 | - |

### 心跳检测机制
- **客户端发送**：每 3 秒 POST `/heartbeat`
- **服务器检测**：每 5 秒检查最后心跳时间
- **超时阈值**：10 秒无心跳 → 标记 `active=false` → 从 `_sessions` 移除

## 数据流

### Defender → Presenter 推流
```
Windows:
  WinScreenCapturer.capture() → JPEG(15fps)
    → POST http://{teacherIp}:8766/frame/win

Android:
  PhoneScreenCapturer (MediaProjection) → JPEG(10fps)
    → POST http://{teacherIp}:8766/frame/phone

Camera (通用):
  LiveStreamService.takeSnapshot() → JPEG(3fps)
    → POST http://{teacherIp}:8766/frame/camera
```

### Presenter → Viewer 流分发
```
DefenseStreamingServer:
  _latestWinFrame / _latestPhoneFrame / _latestCameraFrame
    → 存入内存缓存
    → _broadcast() 遍历 _streamSessions
    → 写入 multipart/x-mixed-replace MJPEG 边界帧
    → HTTP 长连接推送给所有 Viewer
```

## 使用场景

### 场景 1: 学生 Windows 答辩
1. 教师端（Windows）：进入"考核" → "答辩" → 点击"LAN 直播（新）" → 角色"主播" → 点击"开始直播"
2. 教师端：点击"授权管理" → 勾选答辩学生（如 2020211613）→ 保存
3. 学生端（Windows）：收到通知"您已被授权进行答辩直播，教师机IP: 192.168.1.100"
4. 学生端：点击通知"开始答辩"按钮 → 自动跳转到答辩页面（IP 已填充）
5. 学生端：自动启动 Windows 桌面+摄像头推流
6. 观众端：进入"考核" → "答辩" → "LAN 直播（新）" → 输入教师 IP → 连接观看

### 场景 2: 学生 Android 答辩
1-3. 同上
4. 学生端（Android）：点击通知 → 系统弹出"录制屏幕权限"对话框 → 授权
5. 学生端：自动启动 MediaProjection 全屏捕获+摄像头推流
6. 观众端：同上

### 场景 3: 教师手机投屏演示
1. 教师端（Windows）：进入"考核" → "答辩" → "LAN 直播（新）" → 开始直播
2. 教师端（Android）：进入同一页面 → 角色选"手机投屏" → 输入教师机 IP → 点击"投屏"
3. 观众端：看到教师手机屏幕画面

## 已知限制

1. **仅局域网**：不支持公网穿透，需同一 WiFi / 有线网络
2. **端口固定**：8766（可在代码中修改 `_port` 初始值）
3. **单向推流**：Defender → Presenter，不支持双向对讲
4. **Web 平台**：暂未实现屏幕捕获（需 `getDisplayMedia` API）
5. **分辨率**：Windows 15fps 1920x1080，Android 10fps 720x1280，Camera 3fps 640x480
6. **延迟**：LAN 环境约 300-500ms（HTTP 轮询+JPEG 编码）

## 故障排查

| 问题 | 排查方法 |
|------|---------|
| 观众端连接失败 | 1. ping 教师 IP 确认网络连通；2. 检查防火墙是否拦截 8766 端口；3. 教师端确认"开始直播"已启动 |
| 答辩学生推流无画面 | 1. 检查通知 serverIp 是否正确；2. 教师端查看 `/status` 接口 `connectedDevices` 是否有 defender 会话 |
| 教师主播卡死 | 已修复（移除空 `setState` 3秒定时器），若仍有请检查 GPU 占用 |
| Android 黑屏 | MediaProjection 权限未授予，重新授权 |
| 观众数始终为 0 | `/status` 接口返回失败，检查 `_statusTimer` 是否启动 |
