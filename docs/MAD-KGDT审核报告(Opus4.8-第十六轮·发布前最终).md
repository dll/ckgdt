# MAD-KGDT 发布前最终总结审核（第十六轮）

> **本轮性质**：第十五轮是"发布前最终总结审核"，给出**唯一一条 🔴 发布阻断项**——硬编码并公网泄露的 DeepSeek API Key，并明确"发布放行条件：先吊销并清除该 key 再打 release"。本轮（间隔 14 天、跨 4 个 minor 版本：1.17→1.21.2）的第一职责就是核实：**那条红线兑现了吗？**
>
> **结论先行**：**没有。** `sk-717ef9146311424daa2fbead8ed4682b` 仍明文躺在 `ai_config_model.dart:30` + `database_helper.dart:840/1992` 三处，与第十五轮一字不差；并且本轮实测发现**讯飞 STT 的 AppId/ApiKey/ApiSecret 也同样硬编码**在 `settings_service.dart:149-151`（第十五轮未单独点名）。也就是说：**唯一的发布阻断项跨 4 个版本未处理，且同类问题面比上一轮认知的更大。** 除此之外项目继续高速演进（达成度闭环、答辩 LAN 直播、归档教学任务书、课件生成），0 编译错误、版本号 11 字段全对齐 1.21.2，工程主体健康——但**发布放行的硬卡口原地未动，且 CI catch(_) 棘轮已被突破（实际 190 > ceiling 0）**。

所有数字为 2026-06-15 实测（`flutter analyze` / `git grep` / `wc -l`，HEAD @42d6c44fe）。

## 零、🔴 发布前阻断项 — 第十五轮红线兑现核实（**未兑现**）

> 总结审核第一问：**有没有任何东西能让"发布"这个动作本身变成事故？** 第十五轮答过一次，本轮逐字复核。

### 0.1 DeepSeek API Key 硬编码 + 公网泄露 —— 跨 4 版本未处理

实测 `sk-717ef9146311424daa2fbead8ed4682b` 仍明文出现在 **3 处源码**（与第十五轮完全相同）：

| 文件 | 行 | 上下文 | 第十五轮 | 第十六轮 |
|------|----|--------|---------|---------|
| `lib/data/models/ai_config_model.dart` | 30 | provider→key 默认映射 | 🔴 | 🔴 原样 |
| `lib/data/local/database_helper.dart` | 840 | DB 初始化插入默认 AI 配置 | 🔴(670) | 🔴 行号漂移仍在 |
| `lib/data/local/database_helper.dart` | 1992 | 另一路径 | 🔴(1725) | 🔴 行号漂移仍在 |

> 行号从 670/1725 漂到 840/1992 证明这两个文件持续被改动（DB version 27→30），**但每次改动都从这个 key 旁边路过而没动它**——不是"没机会改"，是"没被当回事"。

### 0.2 讯飞语音 AppId/ApiKey/ApiSecret 硬编码（本轮新认定，红线扩面）

```
lib/services/settings_service.dart:149  _defaultXunfeiAppId  = 'ae4a0e4a';
lib/services/settings_service.dart:150  _defaultXunfeiApiKey = '7385e5cb...'（32 hex）;
lib/services/settings_service.dart:151  _defaultXunfeiApiSecret = 'NTI2NzVl...'（base64）;
```

作为"用户未配置时的默认值"兜底注入。与 DeepSeek key 同性质：随公开仓库（Gitee + GitHub Pages）泄露的真实第三方凭证，可被提取后盗用讯飞额度、产生真实账单。第十五轮聚焦 DeepSeek 未点名讯飞——**本轮认定红线面积比上轮认知更大**。

### 0.3 违反的明文规则

CLAUDE.md：「不在代码中硬编码 API Key（默认配置通过 DB 迁移写入）」。设计（配置走 DB / 设置页）是对的，**默认值注入实现破线**——4 处源码、2 个第三方平台凭证。

### 0.4 发布放行条件（与第十五轮一致，仍未满足）

1. **DeepSeek + 讯飞控制台分别吊销/轮换**已泄露的 key（删代码挽不回已泄露的旧 key，必须作废）；
2. 源码 4 处改空值/占位，引导用户在「AI 数据设置」/「语音设置」页填入；
3. 验证：清空后默认无 key，首次进 AI / 语音功能提示去设置页配置，不崩溃。

> **为什么仍是唯一硬卡口**：其余皆为可发布后偿还的慢性债；唯独凭证泄露是"发布动作本身放大伤害"——已跨 4 个版本、14 天，每多公开一天泄露面就再扩一分。**这一轮的核心信息不是"发现新问题"，而是"上一轮拦下的问题没被处理就又要发版了"。**
