# 教学案例技能

## 目标

教学案例技能用于把外部项目目录接入 CKGDT 的“案例”页和“教学案例演示”智能体。它解决三个问题：

- 平台如何识别案例是什么类型的应用。
- 平台如何启动或引导启动该应用。
- 教师如何看到演示步骤、启动截图和案例特色。

## 推荐文件

每个案例项目根目录放置：

```text
教学案例.md
```

平台添加案例时选择项目目录或直接选择该文件，会优先读取模板字段。

## 字段约定

必填或推荐字段：

- `案例名称`
- `应用类型`
- `启动应用的方法`
- `查看应用的步骤`
- `启动后的截图`
- `应用特色内容的介绍`

可选字段：

- `案例全称`
- `项目路径`
- `启动命令`
- `案例简介`
- `仓库地址`

## 运行器规则

- 如果填写 `项目路径`，运行器以该路径作为实际运行目录；相对路径按 `教学案例.md` 所在目录解析。
- 如果填写 `启动命令`，运行器优先执行该命令。
- 如果没有填写启动命令，运行器按项目结构自动识别。
- `启动后的截图` 支持相对路径和绝对路径。

## 自动识别能力

平台会自动识别：

- `.apk`：通过 ADB 安装和启动。
- `.hap`：作为 HarmonyOS HAP 文件打开。
- `.exe`：直接启动 Windows 可执行文件。
- `.bat` / `.cmd`：通过 `cmd /c` 启动。
- Java/Spring Boot：识别 `pom.xml`、`deploy`、`output` 和启动脚本。
- Android/Gradle：识别 `app/src/main/AndroidManifest.xml`，默认执行 `gradlew.bat installDebug`。
- HarmonyOS/ArkTS：识别 `oh-package.json5`、`hvigorfile.ts`、`build-profile.json5`。
- Flutter：执行 `flutter run`。
- Node：执行 `npm start`。

## 智能体用法

选择“🧩 教学案例演示”智能体后，可以问：

- 当前有哪些教学案例？
- 帮我生成案例演示流程。
- 帮我写截图讲解词。
- 这个 APK 案例如何演示？
- 帮我审查案例能不能正常启动。

智能体会优先读取案例库中由 `教学案例.md` 导入的信息，再结合项目类型给出启动步骤和课堂讲解词。
