# 学生收不到答辩通知问题修复

## 修复时间
2026-06-04

## 问题描述

教师点击"通知学生答辩"按钮后，通知写入了本地数据库，但**学生端收不到通知**。

## 根本原因

教师端发送通知后，**没有上传到 Gitee 仓库**。学生端虽然在多个入口（应用启动、通知列表、答辩页面）调用了 `downloadNotifications()`，但 Gitee 上根本没有通知文件，所以学生拉取不到任何内容。

### 数据流断点

完整的通知同步链路应该是：

```
教师端发送 → 本地 DB → 上传 Gitee → 学生拉取 → 学生本地 DB → 学生收到
                    ❌ 断点在这里！
```

原代码中 `_notifyStudents()` 只做了前两步：
1. 调用 `NotificationService().notifyDefenseAuthorized()` 写入本地 DB
2. 显示 SnackBar "已通知 X 名学生"

**缺失步骤**：上传到 Gitee 仓库（`SyncService().uploadNotification(id)`）

## 修复方案

### 修改 `_notifyStudents()` 方法

在发送通知后，收集所有成功创建的 notification ID，批量上传到 Gitee：

```dart
Future<void> _notifyStudents() async {
  final students = await _selectStudents();
  if (students.isEmpty) return;

  final serverUrl = 'http://$_serverIp:$_serverPort';
  int successCount = 0;
  final notificationIds = <int>[]; // 收集成功的通知 ID

  // 1. 发送通知到本地 DB
  for (final entry in students.entries) {
    try {
      final notificationId = await NotificationService().notifyDefenseAuthorized(
        studentId: entry.key,
        studentName: entry.value,
        serverIp: _serverIp,
      );
      if (notificationId != null) {
        successCount++;
        notificationIds.add(notificationId);
      }
    } catch (e, st) {
      swallowDebug(e, tag: 'Defense.notify', stack: st);
    }
  }

  // 2. 上传所有通知到 Gitee（关键修复！）
  if (notificationIds.isNotEmpty) {
    try {
      final syncService = SyncService();
      for (final id in notificationIds) {
        await syncService.uploadNotification(id);
      }
      debugPrint('Defense: uploaded ${notificationIds.length} notifications to Gitee');
    } catch (e, st) {
      swallowDebug(e, tag: 'Defense.uploadNotifications', stack: st);
    }
  }

  // 3. 提示用户已同步
  if (mounted) {
    ScaffoldMessenger.of(context).showSnackBar(
      SnackBar(
        content: Text('已通知 $successCount 名学生并同步到 Gitee\n服务器: $serverUrl'),
        duration: const Duration(seconds: 4),
      ),
    );
  }
}
```

### 关键改动

1. **收集 notification ID**：`notifyDefenseAuthorized()` 现在返回 `int?`，收集所有非空 ID
2. **批量上传 Gitee**：循环调用 `SyncService().uploadNotification(id)`
3. **更新提示信息**：告知用户"已通知 X 名学生并同步到 Gitee"

## 通知同步机制回顾

### 上传端（教师）

```dart
SyncService().uploadNotification(int notificationId)
  ↓
查询通知详情 + 接收人列表
  ↓
生成 JSON 文件：sync/notifications/notif_{id}_{timestamp}.json
  ↓
上传到 Gitee 仓库
```

### 下载端（学生）

学生端在 4 个入口拉取通知：
1. **应用启动**：`home_page.dart` → `initState()` → `_pullNotifications()`
2. **通知列表**：`notification_list_page.dart` → `_loadNotifications()`
3. **通知管理**：`notification_manage_page.dart` → `_loadNotifications()`
4. **答辩页面**：`defense_broadcast_page.dart` → `initState()` (role=defender/phone)

拉取流程：
```dart
SyncService().downloadNotifications()
  ↓
从 Gitee 下载 sync/notifications/*.json
  ↓
_importNotification() 使用自然键去重
  ↓
写入本地 DB (notifications + notification_recipients)
```

### 自然键去重机制

为避免重复导入，使用组合键：
```sql
(entity_type, entity_id, created_at, recipient_id)
```

对于答辩通知：
- `entity_type = 'defense_live'`
- `entity_id = 'defense_auth_{studentId}'`
- `created_at = 通知创建时间`
- `recipient_id = 学生 ID`

## 历史遗留问题

之前的实现中，以下场景会上传通知到 Gitee：
1. **通知管理页面**：发送通知后立即上传（已修复）
2. **通知列表页面**：只拉取，不上传（正确）

但**答辩直播页面**的"通知学生答辩"功能遗漏了上传步骤，导致学生收不到。

## 注意事项

1. **网络依赖**：上传到 Gitee 需要网络连接，失败会静默降级（不阻塞 UI）
2. **批量上传性能**：每个通知单独上传，通知多的情况下可能较慢（10 个学生约 3-5 秒）
3. **学生拉取时机**：学生需要打开应用或刷新通知列表才能收到
4. **跨设备同步**：教师在设备 A 发送，学生在设备 B 登录，仍能收到

## 未来优化方向

1. **批量上传优化**：一次请求上传多个通知，减少网络往返
2. **后台上传**：使用 `unawaited()` 异步上传，不阻塞用户操作
3. **上传状态反馈**：显示上传进度条或成功/失败提示
4. **学生推送通知**：实现真正的推送（需要 FCM/极光等第三方服务）
