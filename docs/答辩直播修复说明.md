# 答辩直播功能修复说明

## 问题描述

2026-06-03 发现答辩直播功能存在两个严重错误：

### 错误 1：URI 格式错误
```
Invalid argument(s): No host specified in URI http://:8766/raw/phone
```
- **位置**：`defense_viewer_widget.dart:68`
- **原因**：`_remoteServerUrl` 为空字符串时，构建的 URL 变成 `http://:8766/raw/phone`，缺少主机名
- **触发场景**：答辩学生（defender）角色初始化时，如果 IP 输入框为空或只有部分输入（如 `192.168.`）

### 错误 2：Disposed Widget 上调用 setState
```
Null check operator used on a null value at State.setState
```
- **位置**：`defense_viewer_widget.dart:86`
- **原因**：异步连接失败时，widget 可能已经 disposed，但错误处理中仍尝试调用 `setState`
- **触发场景**：连接失败后触发重连，但页面已经关闭

## 修复方案

### 1. DefenseViewerWidget (`defense_viewer_widget.dart`)

#### 修复 1.1：URL 验证
在 `_connect()` 方法中添加 URL 格式验证：
```dart
// 验证 URL 格式
Uri? uri;
try {
  uri = Uri.parse(url);
  if (!uri.hasAuthority || uri.host.isEmpty) {
    if (mounted) setState(() { _error = true; _status = 'URL 格式错误'; });
    return;
  }
} catch (e) {
  if (mounted) setState(() { _error = true; _status = 'URL 无效: $e'; });
  return;
}
```

#### 修复 1.2：所有 setState 加 mounted 检查
- `_connect()` 中所有 `setState` 调用前检查 `mounted`
- `_parse()` 中帧更新前检查 `mounted`
- `_schedule()` 中重连前检查 `mounted`

### 2. DefenseBroadcastPage (`defense_broadcast_page.dart`)

#### 修复 2.1：initState 中不自动初始化空 IP
```dart
if (_role == 'defender') {
  // 只有在有有效 IP 的情况下才自动初始化
  if (widget.serverIp != null && widget.serverIp!.isNotEmpty) {
    WidgetsBinding.instance.addPostFrameCallback((_) => _initDefender());
  }
}
```

#### 修复 2.2：_initDefender 中用 setState 更新 URL
```dart
if (mounted) {
  setState(() {
    _remoteServerUrl = 'http://$ip:8766';
  });
}
```

#### 修复 2.3：_buildDefenderPreview 中检查非空且非空字符串
```dart
child: _remoteServerUrl != null && _remoteServerUrl!.isNotEmpty
    ? DefenseViewerWidget(...)
    : _empty('连接中...')
```

#### 修复 2.4：添加手动开始按钮
答辩学生页面在未连接时显示 IP 输入框和"开始答辩"按钮，连接后才显示状态栏和视频预览。

## 修复效果

1. **URL 格式错误已解决**：空 IP 不再触发无效 URL 构建
2. **setState 崩溃已解决**：所有异步操作中的 `setState` 都有 `mounted` 保护
3. **用户体验改善**：答辩学生需要手动输入完整 IP 并点击"开始答辩"，避免了自动连接空 URL 的情况
