# 答辩直播操作手册

## 概述

答辩直播功能支持**四种角色**：
- **Presenter（教师主播）**：运行流媒体服务器，接收并转发所有流
- **Phone（手机投屏）**：学生用手机投屏答辩演示（推荐 Android 学生使用）
- **Defender（答辩学生）**：Windows 学生推送桌面+摄像头到教师服务器
- **Viewer（观众）**：观看答辩直播（非答辩学生+其他教师）

核心流程：**教师开播 → 学生手机投屏/桌面推流 → 观众观看**

**重要变更（v1.19.0）**：
- ✅ 新增"投屏"按钮，学生可直接进入手机投屏模式
- ✅ 移除通知授权机制，学生手动输入教师 IP 即可
- ✅ 所有角色都能通过 AppBar 切换

---

## 一、教师端操作

### 1.1 开启直播服务器

1. 进入 **考核 → 答辩 → LAN 直播（新）**
2. 点击绿色 **主播** 按钮
3. 点击 **开始** 启动服务器
4. 记录显示的服务器 IP（如 `192.168.1.100:8766`）
5. 将 IP 地址告知需要答辩的学生

**注意**：无需"授权答辩学生"，学生直接输入 IP 即可连接

### 1.2 查看直播画面

- 教师端会显示学生推送的画面（手机/桌面 + 摄像头双分屏）
- 实时显示观众数量
- 支持四种布局模式：dual（并排）/ phoneOnly / winOnly / cameraOnly

---

## 二、学生端操作（手机投屏 - 推荐）

### 2.1 进入手机投屏模式

1. 打开"考核" Tab → 找到"LAN 直播（新）"卡片
2. 点击橙色**"投屏"**按钮
3. 输入教师告知的 IP 地址（如 `192.168.1.100`）
4. 点击**"投屏"**按钮

### 2.2 授权屏幕录制（Android）

- Android 系统弹出"屏幕录制"权限对话框
- 点击**"立即开始"**
- 系统提示"已连接"，手机屏幕开始推流到教师机

### 2.3 状态指示

顶部状态栏显示：
- **手机**：绿色勾 = 正在推流（仅 Android）
- **摄像头**：绿色勾 = 正在推流

### 2.4 停止投屏

点击**"停止"**按钮或点击 AppBar 的其他角色 chip 切换角色

---

## 三、学生端操作（桌面推流 - Windows）

### 3.1 进入答辩模式

**方式 1**：通过角色切换
1. 点击"主播"或"观看"按钮进入页面
2. 点击 AppBar 右上角的**"答辩"** chip

**方式 2**：直接访问（需要从代码层面添加入口）

### 3.2 开始答辩推流

1. 输入教师告知的 IP 地址
2. 点击**"开始答辩"**按钮
3. Windows 自动启动桌面捕获（15fps）+ 摄像头（3fps）
4. 推送到教师服务器 `/frame/win` + `/frame/camera`

### 3.3 结束答辩

点击底部红色**"结束答辩"**按钮

---

## 四、观众端操作

### 4.1 手动连接

1. 进入 **考核 → 答辩 → LAN 直播（新）**
2. 点击灰色边框**"观看"**按钮
3. 输入教师机 IP（询问教师获取）
4. 点击 **连接** 按钮
5. 自动播放 MJPEG 流

### 4.2 通过角色切换

- 点击任意按钮进入页面
- 点击 AppBar 右上角的**"观看"** chip
- 输入 IP 并连接

---

## 五、三个入口按钮说明

在"考核 → 答辩 → LAN 直播（新）"卡片中有三个按钮：

| 按钮 | 颜色 | 初始角色 | 适用人员 | 功能 |
|------|------|---------|---------|------|
| **主播** | 绿色 | presenter | 教师 | 启动服务器，接收并预览学生画面 |
| **投屏** | 橙色 | phone | 学生（Android） | 推送手机屏幕到教师服务器 |
| **观看** | 灰色边框 | viewer | 所有人 | 观看答辩直播 |

**角色切换**：进入页面后，可通过 AppBar 右上角的 chip 切换到其他角色

## 五、数据流架构

```
┌─────────────────────────────────────────────────────────────┐
│                     教师端 (Presenter)                        │
│                  DefenseStreamingServer                       │
│                     HTTP 服务器 :8766                         │
│                                                               │
│  接收 ←─────────────────────┐                                │
│  /frame/win      (Windows桌面)                               │
│  /frame/phone    (Android屏幕)                               │
│  /frame/camera   (摄像头)                                     │
│                                                               │
│  分发 ─────────────────────→                                 │
│  /raw/win        (单路Windows)                               │
│  /raw/phone      (单路手机)                                   │
│  /raw/camera     (单路摄像头)                                 │
│  /stream/feed    (混合流)                                     │
└───────────────────────────────────────────────────────────────┘
         ▲                                      │
         │ POST 推流                            │ GET 拉流
         │ 15fps / 10fps / 3fps                 │ MJPEG
         │                                      ▼
┌────────────────────┐              ┌────────────────────┐
│  答辩学生 (Defender) │              │  观众 (Viewer)      │
│                    │              │                    │
│  Windows:          │              │  任意平台           │
│  - 桌面捕获         │              │  DefenseViewer     │
│  - 摄像头           │              │  Widget            │
│                    │              │                    │
│  Android:          │              │  实时 MJPEG 解码    │
│  - MediaProjection │              │  + 心跳保活         │
│  - 摄像头           │              │                    │
└────────────────────┘              └────────────────────┘
```

### 通信端点

| 端点 | 方法 | 用途 | 调用方 |
|------|------|------|--------|
| `/status` | GET | 获取服务器状态 + 观众数 | Presenter |
| `/heartbeat` | POST | 设备心跳（3秒一次） | Defender / Viewer |
| `/sessions` | GET | 查看所有连接设备 | Presenter |
| `/kick/{ip}` | POST | 踢出指定 IP | Presenter |
| `/frame/win` | POST | 推送 Windows 桌面帧 | Defender (Windows) |
| `/frame/phone` | POST | 推送手机屏幕帧 | Defender (Android) |
| `/frame/camera` | POST | 推送摄像头帧 | Defender |
| `/raw/win` | GET | MJPEG 单路 Windows 流 | Viewer |
| `/raw/phone` | GET | MJPEG 单路手机流 | Viewer |
| `/raw/camera` | GET | MJPEG 单路摄像头流 | Viewer |
| `/stream/feed` | GET | MJPEG 混合流（自动选最新） | Viewer |

## 六、帧率与性能

| 源 | 期望帧率 | 分辨率 | 编码 | 带宽 |
|----|---------|--------|------|------|
| Windows 桌面 | 15 FPS | 1280x720 | JPEG Q80 | ~3-5 Mbps |
| Android 屏幕 | 10 FPS | 720x1280 | JPEG Q75 | ~2-4 Mbps |
| 摄像头 | 3 FPS | 640x480 | JPEG Q80 | ~0.5-1 Mbps |

**优化**：
- Windows 桌面捕获使用 `compute()` isolate 异步编码，不阻塞主线程
- 摄像头使用 `takeSnapshotBytes()` 直接返回内存字节，避免文件 I/O
- 状态轮询仅在数据变化时触发 `setState`，减少不必要的重绘

## 七、故障排查

### 7.1 答辩学生看到摄像头但看不到桌面

**症状**：Defender 页面预览只显示右侧摄像头，左侧桌面/手机屏幕是黑屏或"连接中..."

**可能原因**：

1. **Windows 端**：桌面捕获未启动
   - 检查：控制台是否有 `Defender.win` 错误日志
   - 解决：确认 `WinScreenCapturer.initialize()` 被调用，检查 `_winCap.isReady`

2. **Android 端**：屏幕录制权限未授予
   - 检查：状态徽章"手机"是否显示绿色勾
   - 解决：重新点击"开始答辩"，在权限对话框点击"立即开始"

3. **推流失败**：网络问题或教师服务器未启动
   - 检查：教师端是否显示"直播中"状态
   - 检查：`curl http://<教师IP>:8766/status` 是否返回 JSON
   - 解决：教师重启服务器，学生重新进入答辩页面

4. **screenCaptureKey 为 null**（Android）
   - 检查：`main.dart` 是否正确设置 `DefenseBroadcastPage.screenCaptureKey`
   - 解决：确认 `RepaintBoundary` + `GlobalKey` 已包裹主 Widget

**调试命令**（在答辩学生机器上运行）：

```bash
# 测试推流（应返回 {"ok":true,"bytes":...}）
curl -X POST http://<教师IP>:8766/frame/win \
  --data-binary @test.jpg \
  -H "Content-Type: image/jpeg"
```

### 7.2 教师端看到摄像头但看不到学生桌面

**症状**：教师端主播页面预览只显示右侧摄像头，左侧 Win 桌面是黑屏

**可能原因**：

1. **学生端未推流**：学生没有点击"开始答辩"，或推流失败
   - 检查：`curl http://localhost:8766/status` 查看 `frames.win` / `frames.phone` 是否为 true
   - 解决：让学生重新点击"开始答辩"

2. **帧过期**：学生推流中断（网络断开或 App 后台）
   - 检查：`/status` 返回的 `frames.winAge` / `frames.phoneAge` 是否 > 10000ms
   - 解决：学生重新连接网络或回到前台

3. **布局模式错误**：教师选择了"仅手机"布局
   - 检查：控制面板布局选择器是否为"并排"
   - 解决：切换到"并排（dual）"模式

**调试命令**（在教师机器上运行）：

```bash
# 查看服务器状态
curl http://localhost:8766/status | jq

# 查看已连接设备
curl http://localhost:8766/sessions | jq

# 查看单路流是否有数据（应返回 MJPEG 流）
curl http://localhost:8766/raw/win -N --max-time 5
```

### 7.3 观众端黑屏

**可能原因**：
1. 教师未开播
2. IP 输入错误
3. 网络不通（防火墙/跨网段）
4. 学生未推流

**解决**：
1. 确认教师端显示"直播中"绿色状态
2. `ping <教师IP>` 确认网络通
3. `telnet <教师IP> 8766` 确认端口通（Windows 需先启用 telnet 客户端）
4. 学生重新点击"开始答辩"

### 7.4 连接后立即断开

**症状**：点击"连接"后画面闪现几帧然后黑屏

**可能原因**：
1. 心跳超时（10 秒无心跳自动断开）
2. 服务器重启
3. 网络不稳定

**解决**：
1. 检查网络延迟：`ping <教师IP>` 延迟应 < 50ms
2. 观众端会自动重连，等待 3-5 秒
3. 如果仍无法连接，手动刷新页面

### 7.5 帧率低/画面卡顿

**可能原因**：
1. WiFi 信号弱（-70dBm 以下）
2. 同时观看人数过多（> 20 人）
3. 答辩学生设备性能低

**解决**：
1. 切换到 5GHz WiFi 或有线网络
2. 教师切换到"仅桌面"布局减少带宽消耗
3. 降低捕获分辨率（代码层面调整）

## 八、网络要求

### 8.1 推荐配置

| 场景 | 网络 | 延迟 | 带宽 |
|------|------|------|------|
| 教室同 WiFi | 5GHz WiFi | < 50ms | ≥ 20Mbps |
| 有线局域网 | 千兆以太网 | < 10ms | ≥ 100Mbps |
| 跨网段 | ❌ 不推荐 | 延迟高 | 不稳定 |

### 8.2 端口与防火墙

- **端口**：TCP 8766
- **防火墙规则**（Windows 教师机）：
  ```powershell
  # 允许入站连接（管理员权限运行）
  netsh advfirewall firewall add rule name="MAD Defense Streaming" dir=in action=allow protocol=TCP localport=8766
  ```

### 8.3 数据流量估算

- **单个观众**：~3-5 Mbps（双分屏）
- **10 个观众**：~30-50 Mbps（教师上行带宽）
- **20 个观众**：~60-100 Mbps

## 九、安全说明

1. **局域网限定**：服务器仅监听局域网，不对公网开放
2. **无身份认证**：同一局域网内任何人都可连接观看
3. **无加密**：HTTP 明文传输（局域网内可接受）
4. **权限管理**：通过授权列表控制谁可以答辩，但不限制谁可以观看

**生产环境建议**：
- 添加 Token 验证
- 使用 HTTPS（自签名证书）
- 限制 IP 白名单

## 十、常见问题 FAQ

**Q1: 为什么答辩学生看不到自己的桌面画面？**

A: 答辩学生页面的预览显示的是从教师服务器**拉取**的流（`http://<教师IP>:8766/raw/win`），不是本地捕获的画面。如果看不到，说明推流失败或教师服务器未收到帧。检查网络连接和教师端 `/status` 接口。

**Q2: 可以同时有多个答辩学生推流吗？**

A: 当前设计每次只支持一个答辩学生。如果多个学生同时推流，后推流的会覆盖先推流的（服务器只保留 `_latestWinFrame`）。如需支持多路并发，需改造为多流架构。

**Q3: 观众端可以选择只看桌面或只看摄像头吗？**

A: 可以。观众端连接时可以使用不同的 URL：
- `/stream/feed`：混合流（自动切换）
- `/raw/win`：只看桌面
- `/raw/phone`：只看手机屏幕
- `/raw/camera`：只看摄像头

当前 UI 默认连接 `/stream/feed`，可在代码中修改。

**Q4: 可以录制答辩视频吗？**

A: 当前版本不支持录制。如需录制，可使用 OBS Studio 等第三方工具拉取 `http://<教师IP>:8766/stream/feed` 流进行录制。

**Q5: 为什么手机投屏和答辩推流要分开两个 Tab？**

A: 
- **手机投屏（Phone）**：教师用自己的手机演示给学生看（教师 → 学生）
- **答辩推流（Defender）**：答辩学生推自己的屏幕给教师和观众看（学生 → 教师 + 观众）

两者角色不同，所以分开操作。
