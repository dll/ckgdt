# 答辩直播操作手册

## 概述

答辩直播功能三端协作：**教师机（主播端）** — **学生手机（投屏端）** — **观众端（师生观看）**。  
三路独立流（Win 桌面 / 手机全屏 / 摄像头）通过**局域网直连**（HTTP MJPEG）实时传输，不依赖 Gitee 仓库。

| 角色 | 平台 | 操作位置 | 功能 |
|------|------|----------|------|
| 主播（教师） | Windows 桌面 | 考核 → LAN 直播 → 主播 | 开播/截屏/摄像头/设备管理 |
| 投屏（学生） | Android 手机 | 考核 → LAN 直播 → 手机投屏 | 投屏/停止/权限授权 |
| 观看（师生） | 任意端（Web/Win/Android） | 考核 → LAN 直播 → 观看 | 直播流观看/自动重连 |

---

## 一、主播端操作（教师 Windows）

### 1.1 开播

1. 进入 **考核 → LAN 直播（新）** 卡片 → 点击 **主播**
2. 系统自动分配端口（默认 8766），显示服务器地址如 `http://192.168.1.100:8766`
3. 点击 **开始** 按钮启动直播服务器
4. 点击 **桌面** 开启 Win 桌面抓取（仅 Windows 可用，15fps）
5. 点击 **摄像头** 开启 USB 摄像头（10fps）

### 1.2 切换布局

支持三种画面布局：

| 布局 | 说明 |
|------|------|
| 并排（dual） | 桌面 + 摄像头左右分屏 |
| 仅桌面（winOnly） | 全屏显示桌面 |
| 仅手机（phoneOnly） | 全屏显示手机投屏 |

### 1.3 设备管理

- 开播后每隔 3 秒自动刷新已连接设备列表
- 每个设备显示名称、IP、在线时长
- **心跳检测**：设备每 3 秒发送心跳，超过 10 秒无心跳自动标记离线
- 点击设备右侧的 **×** 按钮可踢出该设备
- 主播端面板实时显示 `观看人数` 和 `设备数`

### 1.4 帧率参考

| 源 | 期望 FPS | 说明 |
|----|---------|------|
| Win 桌面 | 15 FPS | GDI BitBlt 捕获 + JPEG 编码 |
| 摄像头 | 10 FPS | USB 摄像头快照 |
| 手机投屏 | 10 FPS | MediaProjection (API 29+) 原生捕获 |

> 实际帧率受局域网质量影响。使用 5GHz WiFi 或千兆有线可获得最佳效果。

### 1.5 停止直播

- 点击红色 **停止** 按钮 → 关闭服务器 + 断开所有连接
- 或直接退出页面（自动停止）

## 二、手机投屏操作（Android 学生）

### 2.1 准备工作

- Android 10 (API 29) 及以上设备
- 与教师机处于**同一局域网**（同一 WiFi）
- 从教师处获取教师机 IP 地址

### 2.2 投屏步骤

1. 进入 **考核 → LAN 直播（新）** 卡片 → 点击 **观看**（而不是主播）
2. 切到 **手机投屏** Tab（顶部选择栏第三个）
3. 输入教师机 IP 地址（如 `192.168.1.100`）
4. 点击 **投屏** 按钮
5. **重要**：系统弹出"屏幕录制"权限对话框，选"立即开始"
6. 成功后状态显示"已连接"（绿色圆点 + 文字）

### 2.3 权限说明

- 首次使用需授权**屏幕录制权限**
- 权限对话框弹出后有 **5 秒超时**：未操作自动降级为应用内投屏（仅捕获 App 本身界面）
- 状态栏提示：**等待权限授权…** 橙色闪烁
- 拒绝权限同样自动降级

### 2.4 停止投屏

- 点击红色 **停止** 按钮
- 或直接退出页面

### 2.5 注意事项

- 确保教师机已先开播（显示"直播中"状态）
- 投屏过程中保持 App 在前台运行
- 锁屏将终止投屏

## 三、观看端操作（师生通用）

### 3.1 观看直播

1. 进入 **考核 → LAN 直播（新）** 卡片 → 点击 **观看**
2. 输入教师机 IP 地址
3. 点击 **连接**
4. 自动播放 MJPEG 流（multipart/x-mixed-replace）
5. 画面右下角显示 `LIVE` 状态 + 实时 FPS

### 3.2 连接异常处理

| 现象 | 原因 | 处理 |
|------|------|------|
| 连接失败 | IP 输入错误或端口不通 | 确认教师机 IP + 端口 8766 |
| 画面停止 | 网络断开或主播断流 | 等待 3 秒自动重连 |
| 画面模糊 | 帧率低或分辨率低 | 检查 WiFi 信号强度 |
| 状态显示 OFFLINE | 主播停止直播 | 联系主播重启 |

### 3.3 自动重连

- 网络断开后自动等待 3 秒重新连接
- 连接恢复后自动续播
- 不需要手动刷新

## 四、网络要求

### 4.1 推荐配置

| 场景 | 网络 | 延迟 | 带宽 |
|------|------|------|------|
| 教室同 WiFi | 5GHz WiFi | < 50ms | ≥ 10Mbps |
| 有线局域网 | 千兆以太网 | < 10ms | ≥ 100Mbps |
| 跨网段 | 不推荐 | 延迟高 | 不稳定 |

### 4.2 端口

- TCP 8766（MJPEG 流 + HTTP API）
- 需确保防火墙允许入站连接

### 4.3 数据流量

- 单路 MJPEG 流约 2-5 Mbps（720p, JPEG quality 80）
- 三路同时约 6-15 Mbps

## 五、架构说明

```
┌─────────────────────┐       ┌──────────────────────┐
│  教师机（主播端）    │       │  Android 手机（投屏端）│
│                     │       │                       │
│  WinScreenCapturer  │       │  PhoneScreenCapturer  │
│  ──→ 桌面 15fps     │       │  ──→ 全屏 10fps       │
│  CameraService      │       │  MediaProjection API  │
│  ──→ 摄像头 10fps   │       │                       │
│                     │       │                       │
│  DefenseStreaming   │       │  HTTP POST            │
│  Server (HTTP 8766) │◄──────│  /frame/phone         │
│                     │       │  /heartbeat           │
│  MJPEG stream       ──────►│                       │
│  /raw/win           │  ◄────┘                       │
│  /raw/phone         │  HTTP GET                     │
│  /raw/camera        │  (观看端/MJPEG)               │
│  /heartbeat         │                               │
│  /sessions          │       ┌──────────────────────┐│
│  /kick/{ip}         │       │  观看端（任意平台）    ││
│                     │       │                       ││
│  Session Mgmt       │──────►│  DefenseViewerWidget  ││
│  (5s cleanup)       │       │  MJPEG 解析 + 心跳    ││
└─────────────────────┘       └──────────────────────┘
```

### 通信协议

| 端点 | 方法 | 用途 |
|------|------|------|
| `/status` | GET | 获取服务器状态 + 会话列表 |
| `/heartbeat` | POST | 设备心跳（JSON: deviceName, source） |
| `/sessions` | GET | 查看当前所有连接设备 |
| `/kick/{ip}` | POST | 踢出指定 IP 设备 |
| `/frame/{source}` | POST | 帧推送（binary JPEG） |
| `/raw/{source}` | GET | MJPEG 流（multipart/x-mixed-replace） |
| `/stream/feed` | GET | 混合流（最新帧自动切换） |

### 心跳机制

- 设备每 **3 秒** 发送一次 POST /heartbeat
- 服务器每 **5 秒** 清理超过 **10 秒** 无心跳的会话
- 被清理的会话从 /sessions 列表中消失，主播端实时更新

## 六、故障排查

### 6.1 投屏无画面

1. 确认教师机已开播（绿色圆点 + 显示"直播中"）
2. 确认 IP 地址正确
3. 检查 Android 系统权限：设置 → 应用 → MAD-KG → 权限 → 允许"屏幕录制"
4. 重启 App 重试

### 6.2 画面卡顿

1. 检查 WiFi 信号强度（建议 -60dBm 以上）
2. 减少同时观看人数
3. 在主播端切换到"仅桌面"减少带宽消耗
4. 关闭其他大流量应用

### 6.3 连接失败

1. 确认防火墙未拦截 8766 端口
2. 在教师机运行 `netstat -an | findstr 8766` 确认端口监听
3. 在手机端 ping 教师机 IP 确认网络通
4. 检查两端是否在同一子网（如 192.168.1.x）

### 6.4 学生离场未断开

- 心跳超时 10 秒后自动清理
- 教师可手动踢出
- 主播停止直播后所有连接立即释放
