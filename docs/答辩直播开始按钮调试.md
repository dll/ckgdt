## 可能的异常模式及诊断

### 1. 按钮完全无响应
**日志特征**：无任何 "Defense: _toggleBroadcast" 日志
**原因**：按钮回调未触发
**检查**：DefenseControlsPanel 的 onToggleBroadcast 参数传递

### 2. 服务器已运行但未触发回调
**日志特征**：
```
Defense: _toggleBroadcast called, _serverReady=false
DefenseStreamingServer: already running, skipping start
[缺失] Defense: onServerReady callback fired
```
**原因**：start() 检测到 _running=true 直接返回，未调用回调
**修复**：在 start() 的 early return 分支也触发回调

### 3. 网络接口未找到
**日志特征**：
```
DefenseStreamingServer: found 0 interfaces
DefenseStreamingServer: no suitable IP found, using 127.0.0.1
```
**原因**：系统网络配置问题或权限不足
**影响**：返回 127.0.0.1，学生无法连接

### 4. 端口被占用
**日志特征**：
```
DefenseStreamingServer: FAILED to bind server after 20 attempts
```
**原因**：端口 8766-8785 全部被占用
**检查**：netstat -ano | findstr "876" 查看占用情况

### 5. 回调触发但 UI 未更新
**日志特征**：
```
Defense: onServerReady callback fired with ip=192.168.1.100, port=8766
```
但界面仍显示"未开播"
**原因**：setState 未执行（mounted=false）或状态未传递到 DefenseControlsPanel
**检查**：_serverReady 变量值和 isBroadcasting 参数传递
