# 课程大纲解析 — 达成度模块

## 一、原来实现的问题

### 1. 确定性解析器脆弱

`AchievementExcelService.parseSyllabus()` / `parseSyllabusBytes()` 用 OOXML 表格结构解析 docx：

- 只认固定表格布局（第 1 列序号、第 3 列权重、第 4 列指标点、第 5 列满分）
- 不同教师的大纲模板差异大（列合并、表头跨行、混合图片标题），解析命中率低
- 解析不到内容时整条目标被抛弃，结果"漏行"或"全空"
- 回到 `_defaultManualObjectiveRows()` 出 4 行空模板，用户被迫手动填

### 2. AI 解析作为辅助而非主解析

- `aiExtractSyllabus(rawText)` 调用大模型从纯文本抽取，结果质量远超表格解析
- 但抽取结果通过 `mergeSyllabusRows(deterministicRows, aiRows)` 与确定性结果合并：AI 只是"补漏"，遇到两者冲突时优先相信表格解析——恰恰是最不可信的
- 用户不可控：AI 抽对了但表格解错了，合出来还是错的

### 3. 智能体交互不可用

- `AchievementAgent` 定义了 `analyze_syllabus_text` / `clarify_objective` / `submit_syllabus` 三个工具
- 但 `AgentChatOverlay` 把 `initialContext` 注入为 `MessageRole.agent` 的 system 消息
- `buildAiMessages()` 只取 `user` / `assistant` 角色，agent 完全看不见上下文
- 用户打开"智能分析"后 agent 说欢迎语，永远不会自动分析

### 4. 课程数据跨课程串扰

- `AchievementDao.getAllBatches()` 没有 course_name WHERE 子句
- 切换课程后批次数据混在一起显示
- 课程目标保存时写入 `course_name` 但读取时不过滤

## 二、设计方案

### 方案一：管理员/教师用「课程目标管理」完整编辑器

> 入口：首页 → 管理(Tab 8) → 第19项「课程目标管理」

**功能清单：**


| 功能      | 实现方式                                                                 |
| ------- | -------------------------------------------------------------------- |
| 课程下拉切换  | `AchievementContext` 单例 + `ValueNotifier` 联动                         |
| 对照表编辑   | 全部字段 inline `TextField`，idx/weight/满分/平时比/实验比/期末比/章节/实验/考核内容         |
| 插入行     | 点击 `[新增行]` → idx 自增，默认值插入                                            |
| 插入列     | 弹窗输入列名 → `extra_columns_json TEXT` 列持久化                              |
| 导入 docx | `AchievementExcelService` 提纯文本 → 打开 agent 浮层                         |
| 智能分析    | `AgentChatOverlay(initialAgentId:'achievement')` → agent 自动分析当前数据    |
| 保存      | `AchievementDao.saveCourseObjectives(courseName, rows)` → 覆盖当前课程所有目标 |


**约束：**

- 严格按 `course_name` 隔离，切换课程未保存时弹确认对话框
- `[导入大纲]` 不再走表格解析器，改为纯文本提取 + agent 主解析
- 智能体可调用 3 个工具完成 analyze→clarify→submit 闭环

### 方案二：普通教师用「达成 → 上传大纲」一键式

> 入口：达成(Tab 7) → 概览 → `[上传大纲]` 按钮

**简化逻辑：**

1. 选 .docx 文件 → `syllabusRawText()` 提取纯文本
2. 直接打开 `AgentChatOverlay(initialAgentId:'achievement', initialContext: rawText)`
3. agent 自动调用 `analyze_syllabus_text` → 显示解析结果表格
4. 用户可在对话中澄清（"目标2权重是0.30" → `clarify_objective`）
5. 用户说"提交" → agent 调用 `submit_syllabus` 落库
6. 关闭浮层时 `_loadBatches()` 自动刷新

**不再做的事：**

- ❌ 不再调用 `parseSyllabusBytes()` 表格解析
- ❌ 不再做置信度评估（高置信自动保存 / 低置信弹对话框）
- ❌ 不再弹 `SyllabusPreviewDialog` 预览对话框

## 三、实现情况

### 3.1 课程隔离基础设施


| 文件                                      | 说明                                                                                                                   |
| --------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| `lib/services/achievement_context.dart` | 单例 `AchievementContext` + `ValueNotifier<String>`                                                                    |
| `lib/data/local/achievement_dao.dart`   | `getAllBatches({String? courseName})` 默认用 `AchievementContext.instance.courseName` 自动过滤；新增 `getAllBatchesUnscoped()` |


### 3.2 课程目标管理页（对照表编辑器）

- `lib/presentation/pages/admin/course_objectives_manage_page.dart` — 约 643 行
- **功能**：课程下拉、对照表编辑、[导入大纲]/[智能分析]/[新增行]/[新增列]/[保存]
- **导入**：`file_picker` 选 docx → `syllabusRawText()` → `AgentChatOverlay` agent 主解析
- **智能分析**：`AgentChatOverlay(initialAgentId:'achievement', initialContext:当前目标文本)`
- **动态列**：`extra_columns_json TEXT`，新增列名存入 JSON，读取时用 `json.decode` 还原

### 3.3 达成概览上传大纲

- `lib/presentation/pages/achievement/tabs/overview_tab.dart` — `_uploadSyllabus()` 改写
- **原流程**：提纯文本 → 表格解析 → AI 解析 → 合并 → 置信度判断 → 预览 → 保存
- **新流程**：提纯文本 → `AgentChatOverlay` agent 全权处理 → 刷新

### 3.4 达成智能体（AchievementAgent）

- `lib/services/agent/agents/achievement_agent.dart`

**3 个大纲分析工具：**


| 工具                      | 参数                          | 行为                            |
| ----------------------- | --------------------------- | ----------------------------- |
| `analyze_syllabus_text` | raw_text, current_course    | AI 从文本提取目标结构，返回表格 + 待澄清项      |
| `clarify_objective`     | objective_idx, field, value | 更新内存中一条目标的字段值                 |
| `submit_syllabus`       | course_name                 | 写入 `course_objectives` 表，清空内存 |


**会话状态变量：**

- `_pendingObjectives: List<Map<String, dynamic>>` — 待提交的解析结果
- `_currentAnalysisCourse: String` — 当前分析的课程名

### 3.5 智能体浮层自动触发

- `lib/presentation/widgets/agent_chat_overlay.dart`

**修复前**：`initialContext` → 注入 `MessageRole.agent` system 消息 → agent 看不见
**修复后**：`Future.microtask(() => _sendMessage(initialContext))` → 作为首条用户消息自动发送 → 触发 agent `handleMessage`

Agent 一打开就收到完整上下文消息，自然调用 `analyze_syllabus_text` 开始分析。

### 3.6 数据库迁移 V32

- `lib/data/local/database_helper.dart`
- 补齐 `course_objectives` 表 `extra_columns_json TEXT` 列
- 版本号从 31 → 32

### 3.7 本次补齐的稳定性修复


| 修复点   | 实现                                                           |
| ----- | ------------------------------------------------------------ |
| 课程隔离  | `getAllBatches()` 默认按课程管理中的激活课程过滤；课程目标管理切换/保存时同步激活当前课程       |
| 旧数据兼容 | 课程目标页、达成概览、DAO、智能体均兼容 `idx/weight/full_mark/ratio` 字段为数字或字符串 |
| 比例规范化 | `20` / `20%` / `0.20` 均归一为 `0.20`，避免显示成 2000%                |
| 提交闭环  | agent `submit_syllabus` 保存后返回 `agent_result`，页面收到后自动刷新       |
| 管理入口  | 教师/管理员可从课程目标页、管理页、语音/智能体子页面路由进入「课程目标管理」                      |
| 旧提示修正 | 移除“课程序号：当前课程”等错误提示，统一使用“当前课程：课程名”                            |

## 四、达成度概览 vs 课程目标的不同定位


| 维度   | 达成度概览（概览 Tab）                | 课程目标管理（管理页第19项）          |
| ---- | ---------------------------- | ------------------------ |
| 用户   | 所有教师（日常使用）                   | 管理员 / 教师（完整配置）           |
| 入口   | 达成(Tab 7) → 概览               | 首页 → 管理(Tab 8) → 第19项    |
| 核心操作 | 上传 docx → agent 解析 → 确认 → 完成 | 对照表编辑器 + AI 分析 + 行/列编辑   |
| 数据源  | 来自 `course_objectives` 表     | 读写 `course_objectives` 表 |
| 批量   | 单次导入一条大纲                     | 管理全部课程的课程目标              |
| 理解   | "一页纸"教师日常——一键导入，快速出结果        | 完整编辑器——调整、对比、分析、补录       |


**两条路径最终都写到同一张表 `course_objectives`，通过 `course_name` 隔离。**

5. `**aiExtractSyllabus()`** 仍作为备选保留在 `AchievementExcelService`，可考虑逐步废弃
