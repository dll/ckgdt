# 答辩通知重复发送问题修复

## 修复时间
2026-06-04

## 问题描述

教师在"选择答辩学生"对话框中无法区分哪些学生已经收到过答辩通知，容易导致重复发送通知给同一学生。

## 修复方案

### 1. 新增查询方法 `_getNotifiedStudents()`

在 `defense_broadcast_page.dart` 中添加方法，查询已收到过答辩通知的学生：

```dart
Future<Set<String>> _getNotifiedStudents(Set<String> studentIds) async {
  final db = await DatabaseHelper().database;
  final result = await db.query(
    'notification_recipients',
    columns: ['recipient_id'],
    where: '''
      notification_id IN (
        SELECT id FROM notifications
        WHERE type = ? AND related_entity_type = ?
      )
      AND recipient_id IN (${studentIds.map((_) => '?').join(',')})
    ''',
    whereArgs: ['defense', 'defense_live', ...studentIds],
    distinct: true,
  );
  return result.map((r) => r['recipient_id'] as String).toSet();
}
```

**查询逻辑**：
1. 从 `notifications` 表中筛选 `type = 'defense'` 且 `related_entity_type = 'defense_live'` 的通知
2. 从 `notification_recipients` 表中查找接收过这些通知的学生
3. 返回学生 ID 的集合（去重）

### 2. 修改 `_selectStudents()` 方法

在显示选择对话框前，先查询已通知的学生：

```dart
// 查询哪些学生已收到过答辩直播通知
final notifiedStudents = await _getNotifiedStudents(allStudents.keys.toSet());

final selected = await showDialog<Map<String, String>>(
  context: context,
  builder: (ctx) => _StudentSelectionDialog(
    students: allStudents,
    notifiedStudents: notifiedStudents, // 传递已通知学生集合
  ),
);
```

### 3. 更新 `_StudentSelectionDialog` 组件

添加 `notifiedStudents` 参数，在列表中显示绿色"已通知"徽章：

```dart
class _StudentSelectionDialog extends StatefulWidget {
  final Map<String, String> students;
  final Set<String> notifiedStudents; // 新增参数
  
  const _StudentSelectionDialog({
    required this.students,
    required this.notifiedStudents,
  });
}
```

### 4. 列表项 UI 更新

在每个学生项右侧添加徽章标识：

```dart
title: Row(
  children: [
    Text(e.value), // 学生姓名
    if (isNotified) ...[

  ],

```

## UI 效果

**修复前**：
- 所有学生显示一样，无法区分是否已通知
- 教师容易重复发送通知

**修复后**：
- 已通知的学生右侧显示绿色徽章：✓ 已通知
- 徽章包含绿色对勾图标 + "已通知"文字
- 绿色半透明背景 + 绿色边框，视觉清晰

## 数据库表结构

涉及的表：
1. **notifications** - 通知主表
   - `id`: 通知 ID
   - `type`: 通知类型（'defense' 表示答辩通知）
   - `related_entity_type`: 关联实体类型（'defense_live' 表示答辩直播）
   
2. **notification_recipients** - 通知接收记录表
   - `notification_id`: 关联 notifications.id
   - `recipient_id`: 接收者（学生）ID

## 注意事项

1. **查询性能**：使用子查询 + IN 操作，对于少量学生（<100）性能足够
2. **状态实时性**：每次打开选择对话框都重新查询，确保状态准确
3. **仍可重复发送**：徽章仅作提示，不阻止教师再次选择已通知的学生（某些场景可能需要重发）
4. **跨设备同步**：通知数据通过 Gitee 同步，其他设备同步后徽章也会显示
