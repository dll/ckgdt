### 问题 2：手机投屏导致应用退出/崩溃

**症状**：
- 学生在手机上点击"投屏"按钮
- 系统弹出屏幕录制权限请求对话框
- 用户拒绝权限后，应用退出或崩溃

**根本原因**：
Android 原生屏幕捕获 (MediaProjection) 需要用户授权。当用户拒绝权限时：
1. 原生端通过 EventChannel 发送 `PERMISSION_DENIED` 错误
2. Flutter 端的错误处理不完善，导致应用异常

**修复内容**：

1. **phone_screen_capturer.dart** - 完善权限拒绝处理
   ```dart
   - 先订阅 EventChannel 再调用 start()，确保能接收错误
   - onError 回调中取消订阅，标记 _useNative = false
   - 权限被拒后自动降级到 RepaintBoundary 捕获（只捕获本应用）
   - 添加详细日志记录权限请求和降级过程
   ```

2. **defense_broadcast_page.dart** - 添加用户提示
   ```dart
   - 投屏启动后延迟 1 秒检查状态
   - 提示用户："如拒绝屏幕录制权限，将只能捕获本应用内容"
   - 添加 try-catch 捕获启动异常并显示错误信息
   ```

**降级策略**：
- **原生全屏捕获**（需权限）：使用 MediaProjection 捕获整个屏幕
- **RepaintBoundary 捕获**（无需权限）：只捕获本应用的 UI 内容

用户拒绝权限后，系统自动切换到第二种模式，**不会崩溃或退出**，只是捕获范围受限。

**新增日志**：
```

// 或
PhoneScreenCapturer: native error - PERMISSION_DENIED
PhoneScreenCapturer: falling back to RepaintBoundary capture
```

## 技术细节

### Android 屏幕录制权限流程

1. Flutter 调用 `MethodChannel.invokeMethod('start')`
2. 原生端调用 `startActivityForResult(createScreenCaptureIntent(), REQUEST_CODE)`
3. 系统弹出权限对话框："MAD-KGDT 将开始截取您的屏幕上显示的所有内容"
4. 用户选择"立即开始"或"取消"
5. `onActivityResult()` 接收结果
6. 通过 `EventChannel` 发送成功/失败消息给 Flutter
7. Flutter 根据结果启动原生捕获或降级到应用内捕获

### 降级机制的实现

**原生全屏捕获**（MediaProjection）：
- 优点：可捕获整个屏幕，包括其他应用、系统 UI
- 缺点：需要用户授权，Android 10+ 支持

**RepaintBoundary 捕获**（Flutter 截图）：
- 优点：无需权限，跨平台支持
- 缺点：只能捕获本应用内的 UI，无法捕获系统通知、其他应用

降级逻辑：
```dart
try {
  await _methodChannel.invokeMethod('start');
  _useNative = true;
  // 订阅事件流，接收帧数据或错误
  _nativeSub = _eventChannel.listen(onData, onError: (e) {
    _useNative = false;  // 标记降级
    _fallbackToRepaintBoundary(fps);  // 启动降级方案
  });
} catch (e) {
  _fallbackToRepaintBoundary(fps);  // 直接降级
}
```

## 已知限制

1. **RepaintBoundary 模式只能捕获应用内 UI**：
   - 学生切换到其他应用时，画面会冻结
   - 无法显示系统通知、状态栏等

2. **首次投屏需要授权**：
   - Android 每次重启应用都需要重新授权
   - iOS 不支持原生全屏捕获（始终使用 RepaintBoundary）

3. **网络延迟**：
   - 局域网环境下延迟约 200-500ms
   - 画质受压缩质量参数影响（当前 75%）

## 未来改进方向

1. **记住用户授权选择**：首次拒绝后，下次自动使用 RepaintBoundary 模式
2. **自适应画质**：根据网络状况动态调整压缩质量和分辨率
3. **多画面布局**：支持画中画、三分屏等更多布局模式
4. **录制功能**：支持教师端录制答辩过程
