# 🏆 达成分析师智能体

## 概述

**达成分析师**（`AchievementAgent`）是 CKGDT 平台的核心智能体，围绕 OBE（成果导向教育）课程达成度评价全流程设计。智能体覆盖"一键生课 → 大纲导入 → 目标管理 → 成绩录入 → 达成计算 → 报告导出"的完整闭环，支持自然语言交互 + 结构化工具调用。

- **Agent ID**: `achievement`
- **名称**: 达成分析师
- **角色**: OBE 专家
- **优先级**: 5
- **AI 依赖**: 是（DeepSeek / 智谱 GLM-4 多 provider）
- **数据源**: `course_objectives` / `achievement_batches` / `achievement_scores`（sqflite 本地 DB）

---

## 核心能力

| 能力 | 实现方式 | 说明 |
|------|----------|------|
| **大纲解析** | AI 提取 + `AchievementExcelService` | 从 docx/md 中识别课程目标对照表，提取权重、满分、指标点 |
| **目标管理** | DB 读写 + AI 辅助 | 课程目标增删改查，权重合计校验，批量导入/导出 |
| **达成计算** | DAO 层 SQL 聚合 | 按 OBE 公式计算各环节达成度：Σ(成绩×环节权重)/满分 |
| **薄弱分析** | `generateImprovementSuggestions` | 识别未达成目标、未达标学生数量、原因分析 |
| **改进建议** | AI + 规则引擎 | 短期（当学期可改进）+ 长期（课程建设层面）措施 |
| **报告导出** | `AchievementDocxService` | Word 报告 + Excel 数据表（含 OOXML 原生图表） |
| **页面导航** | `AgentAction` 导航指令 | 一键跳转课程目标管理、成绩管理、达成概览等页面 |
| **智能问答** | AI 对话（注入 DB 上下文） | 基于当前课程实际数据的免费问答 |

## 页面导航（Agent 触发）

通过 `AgentAction` 机制，智能体可在对话中直接跳转页面：

| 触发词 | 动作类型 | 目标 |
|--------|----------|------|
| 一键生课 / 生成课程 / 课程生成 | `navigate_home` | 首页（右上角菜单生成课程） |
| 课程目标管理 / 编辑目标 | `navigate_sub_page` | 课程目标管理页 |
| 成绩管理 / 录入成绩 | `inner_tab: achievement.成绩管理` | 达成→成绩管理 |
| 达成度概览 / 查看达成 | `inner_tab: achievement.达成度概览` | 达成→达成度概览 |
| 改进建议 / CQI | `inner_tab: achievement.持续改进` | 达成→持续改进 |
| 导出报告 / 生成报告 | `inner_tab: achievement.报告生成` | 达成→报告生成 |

**实现原理**：
```
用户输入 "导出达成度报告"
  → handleMessage() 检测到 (导出|生成).*报告
  → SpecialAgentTools.generateAchievementReport()
  → 生成 DOCX + 返回文件路径
  → AgentAction(type:'inner_tab', params:{page:'achievement', tab:'报告生成'})
  → AgentChatOverlay._handleAction() → NavigationService.requestInnerTab()
  → 关闭浮层 → 切换到"达成"Tab → 切换到"报告生成"子Tab
```

## 用户操作流程

### 流程一：新建课程 + 导入大纲

```
1. 首页 → 右上角 → 一键生课 → 填写课程信息 → 生成
2. 达成 Tab → 课程目标管理 → 导入 Docx 大纲
   （或：打开达成分析师 → 导入大纲 → 粘贴/上传 docx 内容）
3. AI 自动提取课程目标：
   - 目标名称、权重、满分、指标点
   - 平时/实验/期末比例
   - 支撑章节、实验内容、考核要求
4. 确认无误 → 输入"提交"
   → 写入 course_objectives 表
   → 自动创建默认达成度批次
   → 达成概览页自动刷新
```

### 流程二：上传成绩 + 查看达成

```
1. 达成 Tab → 成绩管理 → 下载模板 / 导入 Excel
2. 导入学生平时成绩、实验成绩、期末成绩
3. 系统自动计算各课程目标达成度
4. 达成分析师 → "达成度概览"
   → 查看各目标达成情况、等级、薄弱分析
```

### 流程三：导出报告

```
1. 达成分析师 → "生成报告"
2. 系统生成 Word 报告（含 OOXML 原生 Excel 图表）
3. 报告自动保存到 out/ 目录
4. 可在"达成 → 报告"页预览、导出、打印
```

### 流程四：持续改进

```
1. 达成分析师 → "改进建议" 或 "CQI"
2. AI 分析该批次薄弱目标
3. 输出具体改进措施（短期 + 长期）
4. 教师可参考改进方案调整教学
```

## 技术架构

### 消息处理链（`handleMessage`）

```
用户消息
  ├─ 检测是否为大纲文本（>500chars + 关键词）
  │    → _analyzeSyllabusText() → 提取目标
  ├─ 检测是否为澄清回复（目标\d+…权重|满分|指标点…)
  │    → _clarifyObjective() → 更新字段
  ├─ 待提交状态下 → 取消/重置
  │    → 清空 _pendingObjectives
  ├─ 待提交状态下 → 提交/保存
  │    → _submitSyllabus() → 写入DB + 创建批次 + AgentAction
  ├─ 检测报告生成意图
  │    → SpecialAgentTools.generateAchievementReport() + AgentAction
  ├─ 检测导航意图（一键生课/课程目标管理/成绩管理/…)
  │    → _NavResult(message, AgentAction) → 页面跳转
  └─ AI 兜底
       → 注入课程目标上下文 → safeAiChatWithTools()
```

### 数据流

```
Syllabus(.docx)              StudentScores(.xlsx)
     │                              │
     ▼                              ▼
analyze_syllabus_text()      ScoresTab.importExcel()
     │                              │
     ▼                              ▼
course_objectives (DB)    achievement_scores (DB)
           │                       │
           └───────────┬───────────┘
                       ▼
              calculateClassAverage()
              generateImprovementSuggestions()
                       │
                       ▼
           生成 Word 报告 + Excel 数据
```

## 设计决策

1. **DB 作为权威数据源**：课程目标存储在 `course_objectives`，批次存储在 `achievement_batches`。AI 可查询但不能凭空生成虚假数据。
2. **大纲解析不走 AI 兜底**：先用 `AchievementExcelService` 结构化提取，提取不到才降级到 AI。
3. **提交自动创建批次**：用户提交目标后立即创建默认达成度批次，减少手动操作。
4. **导航动作内建**：智能体直接控制页面跳转（不是提示用户手动操作），提升效率。
5. **OOXML 原生图表**：Word 报告使用 Word 原生 Excel 图表（非 PNG），与学校格式一致。
