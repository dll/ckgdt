# 课程达成度评价系统 — 完整使用手册

## 一、系统概述

课程达成度评价系统是面向教师端的核心功能模块，用于**自动计算课程目标达成度**并**生成符合学校规范的 Word/Excel 评价报告**。

**平台化设计**：系统支持为**任意课程**创建达成度评价批次，不限于特定课程。只需上传该课程的教学大纲，系统即可自动解析课程目标并生成评价报告。

### 默认课程配置（《移动应用开发》）

系统内置《移动应用开发》课程的默认配置，作为示例参考：

| 课程目标 | 指标点 | 权重 | 核心内容 |
|---------|--------|------|---------|
| 目标1 | 1.4 | 15% | 掌握移动应用开发技术体系（原生/混合/跨平台）及主流平台特性 |
| 目标2 | 3.2 | 25% | 运用跨平台开发框架及小程序技术，结合 AI 编程工具与后端 API 交互 |
| 目标3 | 4.2 | 30% | 调研对比多端开发方案，分析不同技术栈在跨设备适配场景中的优劣 |
| 目标4 | 5.1 | 30% | 遵循软件工程规范，使用现代开发工具完成应用测试与优化 |

> **其他课程**：上传大纲后，系统会自动替换为该课程的目标配置，所有计算和报告均基于实际大纲内容生成。

达成度计算公式：
```
课程目标达成度 = Σ(环节达成度 × 环节权重)
环节达成度 = 班级平均分 ÷ 环节满分
总体达成度 = 目标1达成度×0.15 + 目标2达成度×0.25 + 目标3达成度×0.30 + 目标4达成度×0.30
```

预期阈值：**0.60**（达成度 ≥ 0.60 视为达成）

## 二、从零开始的 8 个菜单流程

达成度评价必须按菜单顺序执行。不要在成绩未导入或计算未完成时直接生成报告，否则报告中的达成度和明细可能不完整。

### 01：达成度概览，导入大纲并新建批次

> **位置**：达成度概览 → 右下角 `+`

1. 首次使用时先选择「上传课程大纲」，支持 `.md` 或 `.docx`。
2. 确认系统解析出的课程目标、权重、毕业要求指标点、满分值和支撑章节。
3. 保存大纲后再选择「新建批次」，填写课程名称、班级、学期和批次名称。

> **提示**：大纲数据按课程名存储，同一课程的多个批次可以共享同一份大纲配置。

### 02：成绩管理，下载模板并导入成绩

> **位置**：成绩管理

1. 选择目标批次。
2. 点击「下载模板」，系统会按课程目标生成 Excel 表头。
3. 在模板中填写平时成绩、实验成绩和期末成绩。
4. 点击「导入成绩 Excel」，系统校验学号、姓名、分值范围和重复数据。
5. 确认无误后导入当前批次。

> **注意**：导入成绩只负责写入数据，不要把“导入成绩”和“计算达成度”混在同一步完成。

### 03：计算过程，核对权重并计算达成度

> **位置**：计算过程

1. 选择已导入成绩的批次。
2. 核对课程目标、平时/实验/期末环节和权重关系。
3. 点击「计算达成度」，生成目标达成度、总体达成度和学生个体明细。

### 04：平时达成，检查平时环节结果

> **位置**：平时达成

查看课堂表现、期间测验和课外学习对课程目标的支撑情况，重点确认平时成绩是否导入完整、目标映射是否正确。

### 05：实验达成，检查实验环节结果

> **位置**：实验达成

查看实验 1 到实验 7 的目标得分归集情况，确认实验环节对目标三、目标四等关键目标的支撑是否充分。

### 06：考核达成，检查期末综合考核结果

> **位置**：考核达成

核对项目、小组、个人和答辩成绩分别对应的课程目标，确认期末综合考核分项没有错位或缺失。

### 07：持续改进，形成教学闭环建议

> **位置**：持续改进

根据未达成目标和薄弱环节查看系统生成的改进建议，并记录下一轮教学需要持续跟踪的问题。

### 08：报告生成，生成达成度评价报告

> **位置**：报告生成

1. 选择已完成计算和复核的批次。
2. 先使用 Markdown 预览检查正文、表格和改进建议。
3. 按归档要求导出 Word、Excel 或 PDF。
4. 导出的文件保存到输出目录，点击 Snackbar 可直接打开。

## 三、8 个标签页功能详解

### Tab 1：达成度概览

- 显示所有批次列表（名称、课程、班级、学期、学生人数、状态）
- 常驻显示已导入的课程大纲（课程目标 + 权重 + 指标点）
- 点击批次卡片可查看详情（学生成绩列表 + 达成度结果）

### Tab 2：成绩管理

- 顶部选择批次
- 三个操作按钮：下载模板 / 导入成绩 Excel / 手动添加成绩
- 下方三个子标签：**平时成绩** / **实验成绩** / **期末成绩**
- 每个子标签显示对应明细表，支持编辑单行数据
- 编辑后自动重算批次达成度

#### 平时成绩结构（满分 20 分）

| 项目 | 满分 | 对应目标 | 达成度计算 |
|------|------|---------|-----------|
| 课堂表现 | 20分 | 目标1 | 得分/100 |
| 期间测验 | 30分 | 目标2 | 得分/100 |
| 课外学习 | 50分 | 目标4 | 得分/100 |
| 总评 | — | — | 课堂×0.2 + 测验×0.3 + 课外×0.5 |

> 目标3无平时考核项。

#### 实验成绩结构（满分 30 分）

| 实验 | 满分 | 对应目标 |
|------|------|---------|
| 实验1-2 | 5分+5分 | 目标1 |
| 实验3-4 | 10分+10分 | 目标2 |
| 实验5-6 | 15分+15分 | 目标3 |
| 实验7 | 40分 | 目标4 |

#### 期末成绩结构（满分 50 分）

| 项目 | 满分 | 对应目标 |
|------|------|---------|
| 项目 | 30分 | 目标1 |
| 小组 | 20分 | 目标2 |
| 个人 | 20分 | 目标3 |
| 答辩 | 30分 | 目标4 |

### Tab 3：计算过程

- 展示大纲目标 + 考核结构 + 计算公式
- 班级概览：4 个目标的达成度柱状图 + 雷达图
- 学生明细表：每人 4 个目标达成度 + 综合达成度
- 成绩分布图：直方图展示各目标得分分布
- 一键创建课程满意度问卷（可选）

### Tab 4-6：平时达成 / 实验达成 / 考核达成

- 各环节的班级平均达成度
- 学生明细表（含各分项得分 + 目标达成度）
- 支持按学号/总评升序/降序排序
- 班级平均指标点达成度雷达图

### Tab 7：持续改进

- 上一轮改进措施执行情况
- 本轮改进建议（根据达成度自动生成）

### Tab 8：报告生成

- 选择批次 → 计算达成度 → 选择导出格式
- Markdown 预览：在线阅读完整报告
- Word 导出：生成含 6 张表格的 `.docx` 文件
- Excel 导出：生成含 5 个工作表 + 图表的 `.xlsx` 文件

## 四、Word 报告结构

导出的 Word 报告（`课程目标达成评价报告.docx`）包含以下内容：

| 章节 | 内容 | 对应表格 |
|------|------|---------|
| 一、基本信息 | 课程信息、毕业要求对应关系、评价方式对照表 | 表0~表1 |
| 二、课程考核标准 | 平时/实验/期末评价标准（优秀/良好/合格/不合格） | 表2~表4 |
| 三、达成度计算 | 各目标 × 各环节的达成度计算明细 + 汇总 | 表5 |
| 四、达成结果分析 | 定量分析 + 定性分析 + 持续改进措施 | 表6 |

## 五、Excel 报告结构

导出的 Excel 报告包含 5 个工作表：

| 工作表 | 内容 |
|--------|------|
| 平时成绩 | 每人课堂表现/期间测验/课外学习 + 目标达成度 |
| 实验成绩 | 每人实验1~7得分 + 目标达成度 |
| 期末成绩 | 每人项目/小组/个人/答辩得分 + 目标达成度 |
| 学生个体课程目标达成度 | 每人 4 个目标的平时/实验/期末三环节达成度 |
| 课程目标点达成度 | 目标 × 环节的达成度汇总 + 条形图 + 散点趋势图 |

## 六、常见问题

### Q1：导入 Excel 后提示「未解析到成绩明细表」？

确认 Excel 包含以下工作表之一：
- 平时成绩表（含学号、姓名、课堂表现等列）
- 实验成绩表（含学号、姓名、实验1~7等列）
- 期末成绩表（含学号、姓名、项目/小组/个人/答辩等列）

系统会自动识别表头中的关键词匹配列。

### Q2：达成度计算结果和手动计算不一致？

检查以下配置是否与大纲一致：
- 课程目标权重（默认 0.15/0.25/0.30/0.30）
- 各目标满分值（默认 15/25/30/30）
- 环节权重（平时20%/实验30%/期末50%）

可通过「上传课程大纲」重新同步配置。

### Q3：导出 Word 报告格式不对？

Word 报告基于模板生成。如果学校模板有变更，将新模板放入 `data/达成/` 目录下，系统会自动匹配使用。

### Q4：如何修改已导入的成绩？

在「成绩管理」标签页，选择对应子标签（平时/实验/期末），点击行末的编辑按钮修改。修改后系统自动重算达成度。

### Q5：一个班级能否创建多个批次？

可以。不同学期或不同考核阶段可创建独立批次，数据互不影响。

## 七、数据流向图

```
课程大纲(.md/.docx)
    ↓ AI解析
course_objectives 表（目标/权重/指标点/满分）
    ↓ 配置
AchievementConfig（SSOT）
    ↓
成绩Excel → 解析 → achievement_pingshi_scores
                  → achievement_experiment_scores
                  → achievement_exam_scores
    ↓ 自动合成
achievement_scores 表（4个目标得分 + 总分）
    ↓ 计算
achievement_batches 表（达成度结果 + 状态）
    ↓ 导出
Word(.docx) / Excel(.xlsx) / PDF
```

## 八、注意事项

1. **大纲优先**：权重/满分以 `course_objectives` 表为准，手动导入的成绩按此配置计算
2. **批次独立**：每个批次的计算结果互不影响，可多次计算同一批次
3. **自动重算**：编辑成绩或重新导入后，系统自动重算达成度
4. **问卷数据**：如已创建课程满意度问卷，定性评价分析会自动引用问卷结果
5. **文件位置**：导出的文件保存在系统的输出目录中，可通过 Snackbar 提示直接打开
6. **模板匹配**：Excel 导出时优先使用学校模板（`data/达成/` 目录下），找不到则自动生成
